## Engineering Constraints (from spec and harness)

- Each step exports exactly one function: `run(workspace: Workspace) -> None`
- No step reads from another step's module — all reads go through `workspace.read_artifact()`
- `task`, `method`, `case` are always separate fields in all artifacts; never collapsed into a compound ID
- `coordinate_*` fields in data_manifest/paper_evidence/repo_evidence are free-form strings — no fixed taxonomy
- `blocker.json` is always written by s09; `blocked: false` means clear
- `passed: false` in structural_check only when `missing_unacknowledged` is non-empty
- s12 skip condition: `workspace.blocked AND execution_log.status == "not_attempted"` (skipped only for pre-execution blockers)
- `experience_record.status` is always `"hypothesis"` at P0; no retrieval or injection implemented
- No mock tests (`docs/TESTING_POLICY.md`): tests drive the real `sobench.llm.complete` over real `data/` inputs and **skip with an explicit reason** when the API key or required real data is absent — correctness is never claimed from mocked behavior
